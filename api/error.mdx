---
title: "HTTPower.Error"
description: "Struct representing a transport or middleware error returned by HTTPower."
---

`HTTPower.Error` is the value inside the `{:error, error}` tuple returned when a request cannot be completed due to a network, transport, or middleware failure. It is distinct from an HTTP error response: if the server replies at all (even with a 500), HTTPower returns `{:ok, %HTTPower.Response{}}`. An `HTTPower.Error` means no usable HTTP response was received, or a middleware (circuit breaker, rate limiter, dedup) rejected the request before it was sent.

```elixir
@type reason ::
  atom()
  | {:http_status, integer(), HTTPower.Response.t()}
  | {:feature_error, module(), term()}

@type t :: %HTTPower.Error{
  reason: reason(),
  message: String.t()
}
```

---

## Fields

<ResponseField name="reason" type="atom() | {:http_status, integer(), HTTPower.Response.t()} | {:feature_error, module(), term()}" required>
  A machine-readable identifier for the error. Use this field for programmatic error handling and branching.

  - Plain atoms (e.g., `:timeout`, `:econnrefused`) represent network and transport errors.
  - `{:http_status, status, response}` indicates a status-code-level error where the response is available.
  - `{:feature_error, module, term}` indicates a middleware crash with the offending module and exception.
</ResponseField>

<ResponseField name="message" type="String.t()" required>
  A human-readable description of the error, suitable for logging. Generated by `HTTPower.Error.message/1` from the `reason` value.
</ResponseField>

## Error Catalog

<AccordionGroup>
  <Accordion title=":timeout — Request timeout">
    The request exceeded the configured timeout (default 60 seconds, override with the `timeout` option).

    ```elixir
    {:error, %HTTPower.Error{reason: :timeout, message: "Request timeout"}}
    ```

    **Mitigation:** Increase `timeout` for slow endpoints, or investigate server-side latency.
  </Accordion>

  <Accordion title=":econnrefused — Connection refused">
    The TCP connection was actively refused by the target host. The server is not listening on that port.

    ```elixir
    {:error, %HTTPower.Error{reason: :econnrefused, message: "Connection refused"}}
    ```

    **Mitigation:** Verify the host and port. Check whether the service is running.
  </Accordion>

  <Accordion title=":econnreset — Connection reset">
    The connection was forcibly closed by the remote peer mid-request.

    ```elixir
    {:error, %HTTPower.Error{reason: :econnreset, message: "Connection reset"}}
    ```

    **Mitigation:** Enable `retry_safe: true` to retry on connection resets. Check for load balancer or firewall timeouts.
  </Accordion>

  <Accordion title=":nxdomain — Domain not found">
    DNS resolution failed. The hostname does not exist or cannot be resolved.

    ```elixir
    {:error, %HTTPower.Error{reason: :nxdomain, message: "Domain not found"}}
    ```

    **Mitigation:** Verify the URL hostname. Check DNS configuration and network connectivity.
  </Accordion>

  <Accordion title=":closed — Connection closed">
    The connection was closed before the request completed. Similar to `:econnreset` but initiated more cleanly.

    ```elixir
    {:error, %HTTPower.Error{reason: :closed, message: "Connection closed"}}
    ```

    **Mitigation:** Check keep-alive settings and connection pool configuration.
  </Accordion>

  <Accordion title=":too_many_requests — Too many requests">
    The rate limiter middleware rejected the request because the configured request budget was exhausted.

    ```elixir
    {:error, %HTTPower.Error{reason: :too_many_requests, message: "Too many requests"}}
    ```

    **Mitigation:** Increase the rate limit budget, add backpressure to your callers, or enable request deduplication.
  </Accordion>

  <Accordion title=":service_unavailable — Service unavailable">
    The circuit breaker middleware is open and is rejecting requests to protect the downstream service.

    ```elixir
    {:error, %HTTPower.Error{reason: :service_unavailable, message: "Service unavailable"}}
    ```

    **Mitigation:** The circuit will attempt to close after its configured recovery window. Review the circuit breaker thresholds if it trips too aggressively.
  </Accordion>

  <Accordion title=":dedup_timeout — Request deduplication timeout">
    A duplicate in-flight request was deduplicated but the original request did not complete within the dedup wait timeout.

    ```elixir
    {:error, %HTTPower.Error{reason: :dedup_timeout, message: "Request deduplication timeout"}}
    ```

    **Mitigation:** Increase the dedup timeout or investigate why in-flight requests are taking longer than expected.
  </Accordion>

  <Accordion title=":network_blocked — Blocked in test mode">
    HTTPower is running in test mode (`config :httpower, test_mode: true`) and the request did not include a `:plug` option or an explicit adapter configuration.

    ```elixir
    {:error, %HTTPower.Error{reason: :network_blocked, message: "Network access blocked in test mode"}}
    ```

    **Mitigation:** Use `HTTPower.Test` helpers or pass `plug: {Req.Test, MyModule}` to mock the request in tests.
  </Accordion>
</AccordionGroup>

## Pattern Matching on Errors

```elixir
case HTTPower.get("https://api.example.com/data") do
  {:ok, %{status: status, body: body}} when status in 200..299 ->
    {:ok, body}

  {:ok, %{status: status}} ->
    {:error, {:http, status}}

  {:error, %HTTPower.Error{reason: :timeout}} ->
    {:error, :request_timed_out}

  {:error, %HTTPower.Error{reason: :econnrefused}} ->
    {:error, :service_down}

  {:error, %HTTPower.Error{reason: :service_unavailable}} ->
    # Circuit breaker is open — fail fast rather than queueing
    {:error, :circuit_open}

  {:error, %HTTPower.Error{reason: :network_blocked}} ->
    # Should only happen in tests — missing mock setup
    raise "Missing HTTP mock for this test"

  {:error, %HTTPower.Error{reason: {:feature_error, module, cause}, message: msg}} ->
    # A middleware crashed — surface the details for debugging
    Logger.error("Middleware #{module} failed: #{msg}", cause: inspect(cause))
    {:error, :internal}
end
```

## Full Struct Definition

```elixir
# lib/httpower/error.ex
defmodule HTTPower.Error do
  defstruct [:reason, :message]

  @type reason ::
    atom()
    | {:http_status, integer(), HTTPower.Response.t()}
    | {:feature_error, module(), term()}

  @type t :: %__MODULE__{
    reason: reason(),
    message: String.t()
  }

  @doc false
  def message({:http_status, status, _response}), do: "HTTP #{status} error"
  def message(:timeout),            do: "Request timeout"
  def message(:econnrefused),       do: "Connection refused"
  def message(:econnreset),         do: "Connection reset"
  def message(:nxdomain),           do: "Domain not found"
  def message(:closed),             do: "Connection closed"
  def message(:too_many_requests),  do: "Too many requests"
  def message(:service_unavailable), do: "Service unavailable"
  def message(:dedup_timeout),      do: "Request deduplication timeout"
  def message(reason),              do: inspect(reason)
end
```
