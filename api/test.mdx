---
title: "HTTPower.Test"
description: "Adapter-agnostic HTTP mocking utilities for ExUnit tests."
---

`HTTPower.Test` provides a test mocking interface that works regardless of which adapter (Finch, Req, Tesla) is installed. Write your tests once; they continue to work even if you switch adapters.

Mocking is implemented via `HTTPower.TestInterceptor`, which runs before any adapter and short-circuits the HTTP call when a stub is registered for the calling process.

<Note>
`HTTPower.Test` requires `plug` as a test dependency. The `Plug.Conn` struct is used to represent requests inside stub functions.

```elixir
# mix.exs
defp deps do
  [
    {:httpower, "~> 0.1"},
    {:plug, "~> 1.15", only: :test}
  ]
end
```
</Note>

---

## Public Functions

### `setup/0`

Prepares the current test process to use HTTP mocking. Must be called before `stub/1`. Registers an `on_exit` callback that cleans up the stub after the test finishes.

```elixir
def setup() :: :ok
```

```elixir
defmodule MyApp.ApiClientTest do
  use ExUnit.Case

  setup do
    HTTPower.Test.setup()
  end

  # tests go here
end
```

---

### `stub/1`

Registers a function to handle all HTTP requests made by the current test process. The function receives a `Plug.Conn` struct representing the request and must return a `Plug.Conn` with the response set using a helper (`json/2`, `html/2`, `text/2`).

```elixir
def stub(fun :: (Plug.Conn.t() -> Plug.Conn.t())) :: :ok
```

<ParamField path="fun" type="(Plug.Conn.t() -> Plug.Conn.t())" required>
  A one-arity function that receives a `Plug.Conn` and returns a `Plug.Conn` with the response populated.
</ParamField>

Calling `stub/1` without first calling `setup/0` raises a `RuntimeError` with instructions.

```elixir
HTTPower.Test.stub(fn conn ->
  case {conn.method, conn.request_path} do
    {"GET", "/users"} ->
      HTTPower.Test.json(conn, %{users: ["alice", "bob"]})

    {"POST", "/users"} ->
      HTTPower.Test.json(conn, %{created: true}, status: 201)

    _ ->
      HTTPower.Test.json(conn, %{error: "not found"}, status: 404)
  end
end)
```

### `json/3`

Sets a JSON response on the conn. Encodes `data` with `Jason.encode!/1`, sets `Content-Type: application/json; charset=utf-8`, and returns the conn with the given status.

```elixir
def json(conn :: Plug.Conn.t(), data :: term(), opts :: keyword()) :: Plug.Conn.t()
```

<ParamField path="conn" type="Plug.Conn.t()" required>
  The conn received in the stub function.
</ParamField>

<ParamField path="data" type="term()" required>
  Any JSON-encodable Elixir value (map, list, string, number, boolean, nil).
</ParamField>

<ParamField path="opts" type="keyword()">
  - `:status` — HTTP status code. Default: `200`.
</ParamField>

```elixir
HTTPower.Test.stub(fn conn ->
  HTTPower.Test.json(conn, %{id: 1, name: "Alice"})
end)

HTTPower.Test.stub(fn conn ->
  HTTPower.Test.json(conn, %{error: "unauthorized"}, status: 401)
end)
```

### `transport_error/2`

Simulates a network transport failure. The stub returns normally (no exception is raised in the stub function), but HTTPower returns `{:error, %HTTPower.Error{reason: :test_transport_error}}`.

```elixir
def transport_error(conn :: Plug.Conn.t(), reason :: atom()) :: Plug.Conn.t()
```

<ParamField path="conn" type="Plug.Conn.t()" required>
  The conn received in the stub function.
</ParamField>

<ParamField path="reason" type=":timeout | :closed | :econnrefused | :nxdomain" required>
  The simulated transport failure. Must be one of the supported atoms; any other value raises `ArgumentError`.
</ParamField>

```elixir
HTTPower.Test.stub(fn conn ->
  HTTPower.Test.transport_error(conn, :timeout)
end)

{:error, error} = HTTPower.get("https://api.example.com/slow")
assert error.reason == :test_transport_error
assert error.message =~ "timeout"
```

---

### `allow/2`

Grants a pre-existing process access to the current test's stub. Use this for GenServers or other processes that are started before the test runs and cannot inherit the caller chain automatically.

Processes **spawned from within a test** (via `Task.async`, `spawn`, etc.) do not need `allow/2` — they automatically find the parent's stub by walking the `$callers` process chain.

Allowances are transitive: if you allow a GenServer that spawns a Task, the Task also sees the stub.

```elixir
def allow(pid :: pid(), owner :: pid()) :: :ok
```

<ParamField path="pid" type="pid()" required>
  The PID of the process to grant access to.
</ParamField>

<ParamField path="owner" type="pid()">
  The test process that owns the stub. Defaults to `self()`.
</ParamField>

```elixir
setup do
  HTTPower.Test.setup()
  HTTPower.Test.allow(Process.whereis(MyApp.HttpWorker))
end

test "worker uses mock" do
  HTTPower.Test.stub(fn conn ->
    HTTPower.Test.json(conn, %{ok: true})
  end)

  assert {:ok, response} = MyApp.HttpWorker.fetch()
  assert response.body["ok"] == true
end
```

---

## How Interception Works

All built-in adapters (Finch, Req, Tesla) call `HTTPower.TestInterceptor.intercept/4` as their first action. The interceptor checks whether the calling process (or any process in the `$callers` chain) has a registered stub:

- If a stub is found — it executes the stub, builds a `Plug.Conn`, runs the stub function, and converts the result to `{:ok, %HTTPower.Response{}}` or `{:error, %HTTPower.Error{}}`.
- If no stub is found — returns `:continue` and the adapter proceeds with the real HTTP request.

This means test mocking does not require any special adapter configuration or environment flags.

## Complete Example

```elixir
defmodule MyApp.WeatherClientTest do
  use ExUnit.Case

  setup do
    HTTPower.Test.setup()
  end

  test "returns current weather" do
    HTTPower.Test.stub(fn conn ->
      case {conn.method, conn.request_path} do
        {"GET", "/weather"} ->
          HTTPower.Test.json(conn, %{
            temperature: 22,
            condition: "sunny",
            city: conn.query_string |> URI.decode_query() |> Map.get("city")
          })

        _ ->
          HTTPower.Test.json(conn, %{error: "not found"}, status: 404)
      end
    end)

    {:ok, response} = HTTPower.get("https://api.weather.com/weather?city=London")

    assert response.status == 200
    assert response.body["temperature"] == 22
    assert response.body["city"] == "London"
  end

  test "handles network timeout" do
    HTTPower.Test.stub(fn conn ->
      HTTPower.Test.transport_error(conn, :timeout)
    end)

    {:error, error} = HTTPower.get("https://api.weather.com/weather")

    assert error.reason == :test_transport_error
  end

  test "handles 429 rate limit response" do
    HTTPower.Test.stub(fn conn ->
      conn
      |> Plug.Conn.put_resp_header("retry-after", "60")
      |> HTTPower.Test.json(%{error: "too many requests"}, status: 429)
    end)

    {:ok, response} = HTTPower.get("https://api.weather.com/weather")
    assert response.status == 429
  end
end
```
