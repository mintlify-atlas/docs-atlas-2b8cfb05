description: "Get HTTPower installed and making your first HTTP request in minutes."
---

<Steps>
  <Step title="Install HTTPower">
    Add `httpower` and at least one HTTP client adapter to your `mix.exs` dependencies:

    <Tabs>
      <Tab title="Finch (recommended)">
        ```elixir mix.exs
        def deps do
          [
            {:httpower, "~> 0.20.0"},
            {:finch, ">= 0.19.0"}
          ]
        end
        ```

        Finch is the default and recommended adapter. It provides high-performance HTTP built on Mint with explicit connection pooling.
      </Tab>
      <Tab title="Req">
        ```elixir mix.exs
        def deps do
          [
            {:httpower, "~> 0.20.0"},
            {:req, ">= 0.4.0"}
          ]
        end
        ```

        Req is a batteries-included HTTP client with automatic JSON handling.
      </Tab>
      <Tab title="Tesla">
        ```elixir mix.exs
        def deps do
          [
            {:httpower, "~> 0.20.0"},
            {:tesla, ">= 1.10.0"}
          ]
        end
        ```

        Use Tesla if you already have a Tesla setup with custom middleware you want to preserve.
      </Tab>
      <Tab title="All adapters">
        ```elixir mix.exs
        def deps do
          [
            {:httpower, "~> 0.20.0"},
            {:finch, ">= 0.19.0"},   # Recommended - high performance
            {:req, ">= 0.4.0"},       # Batteries-included with auto-JSON
            {:tesla, ">= 1.10.0"}     # If you already use Tesla
          ]
        end
        ```

        When multiple adapters are present, HTTPower automatically selects Finch by default (Finch → Req → Tesla priority).
      </Tab>
    </Tabs>

    <Note>
      At least one adapter is required. If multiple are installed, Finch takes priority. You can override the adapter globally in config or per-client/request.
    </Note>

    Then fetch dependencies:

    ```bash
    mix deps.get
    ```
  </Step>

  <Step title="Make your first request">
    HTTPower works immediately with no configuration — just call the functions:

    ```elixir
    # Simple GET request
    {:ok, response} = HTTPower.get("https://api.example.com/users")
    IO.inspect(response.status)  # 200
    IO.inspect(response.body)    # %{"users" => [...]}

    # POST with JSON body
    {:ok, response} = HTTPower.post("https://api.example.com/users",
      body: Jason.encode!(%{name: "John", email: "john@example.com"}),
      headers: %{"Content-Type" => "application/json"}
    )

    # POST with form data
    {:ok, response} = HTTPower.post("https://api.example.com/users",
      body: "name=John&email=john@example.com",
      headers: %{"Content-Type" => "application/x-www-form-urlencoded"}
    )
    ```

    <Note>
      `{:ok, response}` means the server responded — not that the business logic succeeded. This includes 4xx and 5xx responses. Always check `response.status` to determine the outcome.
    </Note>
  </Step>

  <Step title="Create a configured client">
    For multiple requests to the same API, create a reusable client with `HTTPower.new/1`:

    ```elixir
    # Create a client with base URL and shared settings
    client = HTTPower.new(
      base_url: "https://api.example.com",
      headers: %{"authorization" => "Bearer #{token}"},
      timeout: 30,
      max_retries: 3
    )

    # Reuse the client for multiple requests
    {:ok, users}   = HTTPower.get(client, "/users")
    {:ok, user}    = HTTPower.get(client, "/users/123")
    {:ok, created} = HTTPower.post(client, "/users", body: data)
    {:ok, deleted} = HTTPower.delete(client, "/users/456")
    ```

    For common use cases, use a pre-built profile instead of configuring everything manually:

    ```elixir
    # Payment processing: conservative rate limits, dedup to prevent double charges
    payment_client = HTTPower.new(
      base_url: "https://api.stripe.com",
      profile: :payment_processing
    )

    # High-volume API: adaptive rate limiting, fast circuit recovery
    api_client = HTTPower.new(
      base_url: "https://api.github.com",
      profile: :high_volume_api
    )

    # Microservices: prevent cascade failures and retry storms
    service_client = HTTPower.new(
      base_url: "https://internal-service.svc",
      profile: :microservices_mesh
    )
    ```

    <Tip>
      Profile settings can be overridden with explicit options. For example: `HTTPower.new(profile: :payment_processing, max_retries: 5)` uses the payment profile but with 5 retries instead of the profile default of 3.
    </Tip>
  </Step>

  <Step title="Handle errors gracefully">
    HTTPower never raises exceptions for HTTP operations. Every call returns either `{:ok, response}` or `{:error, error}`:

    ```elixir
    case HTTPower.get("https://api.example.com/users") do
      {:ok, %HTTPower.Response{status: status, body: body}} when status in 200..299 ->
        # Success - process the response body
        process_data(body)

      {:ok, %HTTPower.Response{status: 404}} ->
        # Server responded with 404 - handle not found
        handle_not_found()

      {:ok, %HTTPower.Response{status: status}} when status >= 500 ->
        # Server error - after retries exhausted, still returns {:ok, response}
        handle_server_error(status)

      {:error, %HTTPower.Error{reason: :timeout}} ->
        # Network timeout
        handle_timeout()

      {:error, %HTTPower.Error{reason: :econnrefused}} ->
        # Connection refused
        handle_connection_error()

      {:error, %HTTPower.Error{reason: :service_unavailable}} ->
        # Circuit breaker is open
        use_fallback()

      {:error, %HTTPower.Error{reason: :too_many_requests}} ->
        # Rate limit exceeded with :error strategy
        handle_rate_limit()
    end
    ```

    <Warning>
      `{:ok, response}` is returned even for 4xx and 5xx status codes — it means the server responded successfully over the network. Only `{:error, error}` indicates a transport-level failure (timeout, connection refused, etc.) or middleware rejection (circuit open, rate limit exceeded with `:error` strategy).
    </Warning>
  </Step>
</Steps>
